How this App Works

This page is a Web Application built on the Angular framework. When you first request to the load the page, none of the content below exists yet. Instead, a tiny but full-fledge application boots up that has enough brains to do everything it needs from a single page.

Angular app is built of one or more, discrete components. This app happens to have three: one for loading the front page, one for retrieving the sections of the proposal, and one for displaying those sections. Each component does one and only one job; this strong seperation of concerns makes refactoring and extending the app easy.

To replicate an API, the app queries a "database" (just a flat file for now) via HTTP to retrieve an arbitrary number of "section" objects. The data model is defined in such a way that each section has multiple pieces, and each piece can be of several types (header, list item, image, or paragraph). Using this data model and logic defined as part of the section-viewer component, the app renders each type of item differently to dynamically create the document you see below. This is simple templating and one-way data binding, but Angular supports powerful two-way data binding that makes sleek forms a breeze.
